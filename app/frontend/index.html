🧩

Fleet security token

This shared credential lets your Hubs and Studios trust requests from your fleet while rejecting other devices. Save it on the primary Hub to securely synchronize and verify every connected machine.

How fleet authentication and repair work
  • What it protects: remote jobs, status changes, lifecycle controls, settings, uploads, and output access. Local loopback remains passwordless.
  • Where it comes from: the first Hub generates one owner-only local value. The primary Hub is the source of truth; use the same value on every agent Hub.
  • Test or rotate: click Save & verify. Connected peers are updated with the previously trusted value, then verified with the new one. The old value stops working immediately.
  • Repair a mismatch: open the affected Mac's Hub locally, paste the primary Hub's value here, click Save & verify, then restart or update the affected Studio.
  • The value stays in owner-only local files and request headers. It is never included in URLs, logs, screenshots, or fleet diagnostics.
